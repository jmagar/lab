On this page
OpenACP comes with powerful features beyond basic chat. These capabilities let you expose local servers to the internet, resume past sessions with full conversation history, track API spending, and diagnose problems — all from your messaging app or terminal.
##
[](#whats-in-this-section)
What's in this section
*
[**Tunneling**](/features/tunnel) — Expose local ports to the internet via Cloudflare, ngrok, bore, or Tailscale, with a built-in file viewer for code and diffs.
*
[**Context & Resume**](/features/context-resume) — Resume sessions with full conversation history from previous agent runs, with built-in history recording and Entire.io checkpoint support.
*
[**Output Modes**](/features/output-modes) — Choose how much detail you see when your agent is working: just the result, the highlights, or everything.
*
[**Usage & Budget**](/features/usage-and-budget) — Track token consumption and API costs per session, with configurable monthly budget warnings.
*
[**Session Persistence**](/features/session-persistence) — Sessions survive daemon restarts; resume exactly where you left off.
*
[**Session Handoff**](/features/session-handoff) — Transfer a running session from your terminal into Telegram or Discord (and back).
*
[**Agent Switch**](/features/agent-switch) — Switch between AI agents mid-conversation while preserving context.
*
[**Workspaces**](/features/multi-instance) — Run separate OpenACP environments for different projects, teams, or use cases.
*
[**App Connectivity**](/features/app-connectivity) — Connect desktop and web apps to OpenACP via local discovery or remote tunnel links.
*
[**Doctor**](/features/doctor) — Run a full system health check with `openacp doctor` to catch misconfigurations before they cause problems.
*
[**Assistant Mode**](/features/assistant-mode) — A dedicated Assistant topic in Telegram that manages your sessions, agents, and settings through natural language.
