# Messages
Build Assistants that can call models and use tools.
#### resource openai\_beta\_thread\_message
##### required Expand Collapse
thread\_id: String
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) thread_id>)
content: String
The text contents of the message.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) content>)
role: String
The role of the entity that is creating the message. Allowed values include:
* `user`: Indicates the message is sent by an actual user and should be used in most cases to represent user-generated messages.
* `assistant`: Indicates the message is generated by the assistant. Use this value to insert messages from the assistant into the conversation.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) role>)
##### optional Expand Collapse
attachments?: List[Attributes]
A list of files attached to the message, and the tools they should be added to.
file\_id?: String
The ID of the file to attach to the message.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) attachments > (attribute) file_id>)
tools?: List[Attributes]
The tools to add this file to.
type: String
The type of tool being defined: `code\_interpreter`
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) attachments > (attribute) tools > (attribute) type>)
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) attachments > (attribute) tools>)
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) attachments>)
metadata?: Map[String]
Set of 16 key-value pairs that can be attached to an object. This can be
useful for storing additional information about the object in a structured
format, and querying for objects via API or the dashboard.
Keys are strings with a maximum length of 64 characters. Values are strings
with a maximum length of 512 characters.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) metadata>)
##### computed Expand Collapse
id: String
The identifier, which can be referenced in API endpoints.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) id>)
assistant\_id: String
If applicable, the ID of the [assistant](https://platform.openai.com/docs/api-reference/assistants) that authored this message.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) assistant_id>)
completed\_at: Int64
The Unix timestamp (in seconds) for when the message was completed.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) completed_at>)
created\_at: Int64
The Unix timestamp (in seconds) for when the message was created.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) created_at>)
incomplete\_at: Int64
The Unix timestamp (in seconds) for when the message was marked as incomplete.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) incomplete_at>)
object: String
The object type, which is always `thread.message`.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) object>)
run\_id: String
The ID of the [run](https://platform.openai.com/docs/api-reference/runs) associated with the creation of this message. Value is `null` when messages are created manually using the create message or create thread endpoints.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) run_id>)
status: String
The status of the message, which can be either `in\_progress`, `incomplete`, or `completed`.
[](<#(resource) beta.threads.messages > (terraform resource) > (attribute) status>)
incomplete\_details: Attributes
On an incomplete message, details about why the message is incomplete.
reason: String
The reason the message is incomplete.
